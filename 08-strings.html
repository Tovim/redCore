1. Funkce pro řetězcové typy řad

Mezi takzvané řetězcové typy řad patří typy string!, binary!, tag!, file!, URL!, email! a image!.

Pro řetězcové typy řad se používají funkce, definované pro pseudotyp series!. Specifické pro řetězcové typy řad (řetězce) jsou funkce pad, split, trim, uppercase a lowercase.


2. Konverze hodnot na řetězce

2.1 Rejoin

Funkce rejoin redukuje blok vypuštěním mezer a konvertuje jej na řetězec.

rejoin ["try" 1 2 3]
"try123"
rejoin ["h" 'e #"l" (to-char 108) "o"]
"hello"

Typ vrácené řady závisí na hodnotě prvního argumentu. Je-li první argument supertypu series!, tentýž typ řady je vrácen.

str: "abc "
file: %file
url: http://www.rebol.com/

rejoin [str [1 2 3]]
== "abc 1 2 3"
rejoin [file ".txt"]
== %file.txt
rejoin [url %index.html]
== http://www.rebol.com/index.html

Nepatří-li první element bloku k supertypu series!, převede jej funkce rejoin nejprve na string! a poté provede připojení:

rejoin [9:11:01 " elapsed"]
== "9:11:01 elapsed"
rejoin [11:09:11 "AM" " on " now/date]
== "11:09:11AM on 17-Aug-2017"
rejoin [now/date " -- today"]
== "17-Aug-2017 -- today"
rejoin [255.255.255.0 " netmask"]
== "255.255.255.0 netmask"
rejoin [412.452 " light-years away"]
== "412.452 light-years away"
rejoin [312.423 123 987 234]
== "312.423123987234"

Je-li druhý element bloku také blok, provede se nejprve jeho vyhodnocení a poté připojení k prvnímu elementu.

rejoin ["a" ["b" "c" 1 2]]
== "ab c 1 2"
rejoin [%/ [%dir1/ %sub-dir/ %filename ".txt"]]
== %/dir1/%20sub-dir/%20filename%20.txt
Zde je vhodnější použít:
rejoin [%/ %dir1/ %sub-dir/ %filename ".txt"]
== %/dir1/sub-dir/filename.txt

2.2 Form

Funkce form odstraní případné dekorace a přemění hodnotu na řetězec. Výstup z této funkce není vždy vhodný jako argument dalších procesů:

form [:aso /5 "yes"]
"aso 5 yes"

Znaky řetězce odstraní funkce print:
print form [:aso /5 "yes"]
aso 5 yes 

type? form 25
string!

Následující příklad pužívá funkci form k nalezení čísla podle jeho desetinné hodnoty:

blk: [11.22 44.11 11.33 11.11]
foreach num blk [if find form num ".11" [print num]]
44.11
11.11

2.3 Mold

Funkce mold přemění zadanou hodnotu na řetězec, bezpečně použitelný jako argument pro případné další procesy:

mold [:aso /5 "yes"]
{[:aso /5 "yes"]}

Znaky řetězce odstraní funkce print:
print mold [:aso /5 "yes"]
[:aso /5 "yes"] 

type? mold 25
string!

Jednou vidět je lepší než 10x číst:

blk: [[11 * 4] (15 - 3.89) "eleven dollars"]    viz Pozn:
== [[11 * 4] (15.00 - 3.89) "eleven dollars"]

molded-blk: mold blk
== {[[11 * 4] (15.00 - 3.89) "eleven dollars"]}

new-blk: load molded-blk
== [[11 * 4] (15.00 - 3.89) "eleven dollars"]

type? blk
== block!
type? molded-blk
== string!
type? new-blk
== block!

first blk
[11 * 4]
first molded-blk
#"["

Pozn: Výrazy uvnitř bloku se samy od sebe nevyhodnotí ani s funkcí mold.


2.4 Mezery v řetězci

2.4.1 Pad

Funkce pad doplní řetězec na pravé straně mezerami na zadanou délku.

pad "ab" 4                   ; číslo udává výslednou délku řetězce
"ab  "
pad/left " ab" 4
"  ab"

prin "X"  prin pad/left "AB" 6  print "X" print "-123456-"
X   ABX
-123456_

2.4.2 Trim

Funkce trim odebere přebytečné mezery z čela a konce řetězce. S upřesněním /lines odebere všechny přebytečné mezery. Jde o trvalou úpravu proměnné:

str: "  text s přebytečnými    mezerami  "
print trim/lines str
"text s přebytečnými mezerami"

Pro specifikaci místa k odebrání mezery lze použít tato upřesnění:

 /headodebere mezeru jen z čela řetězce
 /tailodebere mezeru z chvostu řetězce
 /linesodebere přebytečné mezery, včetně přerušení řádků (linebreaks)
 /allodebere všechny mezery, nikoliv však přerušení řádků
 /withodebere všechny určené znaky
 /autoodebere mezeru z každého přilehlého řádku    dosud nezavedeno

Použití /with k odebrání zadaných znaků: 

str: "some    extra-characters,, to use"
== "some    extra-characters,, to use"

trim/with copy str ",-"              hodnotu str budeme ještě potřebovat
"some    extracharacters to use"
trim/with copy str ",- "             do parametru přidána mezera!
"someextracharacterstouse"

Použití /all k odebrání všech mezer:

trim/all copy str
"someextracharacterstouse"

2.4.3 Split

Funkce split rozdělí řetězec v označených místech:

str: "Několik koblih"
split str "k"
["Ně" "oli" " " "oblih"]

2.5 Uppercase a lowercase

Funkce pro změnu velikosti písmen: uppercase (verzálky) a lowercase (mínusky). Funkce uppercase přijímá řetězec a přemění jeho znaky na velká písmena:

print uppercase "SamPle TExT, tO test CASES"
SAMPLE TEXT, TO TEST CASES

Funkce lowercase konvertuje znaky na malá písmena:

print lowercase "Sample TEXT, tO teST Cases"
sample text, to test cases

Ke změně pouze části řetězce použijeme upřesnění /part:

print upppercase/part "ukiah" 1
Ukiah

2.6 Checksum

Funkce checksum vrátí kontrolní součet zadaného argumentu typu binary!, string! nebo file!.

Syntaxe:checksumargumentmetoda

Metod pro určení kontrolního součtu je celá řada:

 MD5je 128-bitová číselná hodnota
 CRC32(Cyclic Redundancy Check) je hodnota, určená polynomiálním dělením; číslo za zkratkou určuje stupeň řídícího polynomu
 SHA {1, 256, 384, 512}(Secure Hash Algorithm) jsou {160, 256, 384 a 512}-bitové kryptografické hodnoty
 TCP(Transmission Control Protocol) je 16-bitová číselná hodnota 
checksum "čičmunda" 'md5
== #{21475F7C0CB50ABFD5863CB203576BDB}

checksum "čičmunda" 'crc32
== 338877614

checksum "čičmunda" 'sha1
== #{0AAFE9CD6E52E246093C079C415BE6D45BD3A15F}

checksum "čičmunda" 'sha256
== #{CBBB25D3C8784939D3B52C81AFC45BA7844A3030ECD4D67A693A26BB524FA03...}

checksum "čičmunda" 'tcp
== 27248

Funkce checksum zná upřesnění /with ...


2.7 Enbase a debase

Funkce enbase provede konverzi zadané hodnoty typu binary! a string! na typ string! s tetrahexagesimálním (implicitně), hexadecimálním nebo binárním obsahem.

Syntaxe:enbase[/base]argumentbáze (64, 16, 2) 
line: "No! There's a land!"
== "No! There's a land!"

a-line: enbase line                   ; implicitně: báze 64
== "Tm8hIFRoZXJlJ3MgYSBsYW5kIQ=="
b-line: enbase/base line 64
== "Tm8hIFRoZXJlJ3MgYSBsYW5kIQ=="
c-line: enbase/base line 16
== "4E6F2120546865726527732061206C616E6421"
d-line: enbase/base line 2
== "01001110011011110010000100100000010101000..."
type? b-line
== string!

Zakódované řetězce lze dekódovat funkcí debase. Mějte na paměti, že výsledek je binární hodnota. Pro její konverzi zpět na řetězec bude potřebná funkce to-string.

aa-line: debase a-line
#{4E6F2120546865726527732061206C616E6421}
ab-line: debase b-line
#{4E6F2120546865726527732061206C616E6421}
ac-line: debase c-line
#{E04E85DB5DB4E78EBCEB9EF6EB9DBBEF7DB4EB5DB4E82EB8E84EB8}
ad-line: debase d-line
#{D35D34D75D74D35D74D75D75D34D74D34D35D34D74D34D34D35D35D35...}
type? ad-line
== binary!

to-string ab-line
== "No! There´s a land!"
to-string ac-line                    ; nutno vysvětlit ...
*** Access Error: invalid UTF-8 encoding: #{E04E85DB}

Upřesnění /base n (n = 2, 16, 64) může být použito s funkcemi enbase i debase.


2.8 Hexadecimální dekódování internetu

Funkce dehex konvertuje URL internetu a znaky kódované v hexadecimálním CGI stylu na řetězce. hexadecimal encoded characters to strings. Hexadecimální prezentace ASCII se v řetězcích URL nebo CGI objeví jako %xx, kde xx je hexadecimální hodnota.

str: "there%20seems%20to%20be%20no%20spaces"
dehex str
there seems to be no spaces
dehex "%68%65%6C%6C%6F"
hello
